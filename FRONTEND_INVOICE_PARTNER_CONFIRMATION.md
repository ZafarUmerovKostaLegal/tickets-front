# Инструкция для фронта: счета и подтверждение партнёров

На бэкенде включена связка **необиллированные позиции → счёт** с обязательным **полным подтверждением партнёров** (`fully_confirmed`) для того же **проекта** и **периода** (`dateFrom`–`dateTo`), что при отправке отчёта на подпись партнёрам.

Иначе: не загрузятся строки для выставления счёта, создание счёта с временем/расходами будет отклонено, список счетов в «режиме биллинга» можно показывать пустым по флагу ответа.

---

## 1. Обязательные поля при создании счёта (`POST …/invoices`)

Если в теле есть хотя бы одно из:

- **`timeEntryIds`** (не пустой),
- **`expenseIds`** (не пустой),
- **`lines`** (не пустой массив) **и** задан **`projectId`**,

то **обязательно** передать:

| Поле (camelCase) | Описание |
|------------------|----------|
| `partnerBillingPeriodFrom` | Дата начала периода (ISO `YYYY-MM-DD`), **совпадает** с периодом подтверждённого отчёта |
| `partnerBillingPeriodTo` | Дата конца периода, **совпадает** с подтверждением |

Они должны в точности совпасть с `dateFrom` / `dateTo` у записи подтверждения партнёров со статусом **полного** подтверждения по этому проекту.

**Ответы при ошибках:**

- **400** — не указаны периоды или указан только контент без `projectId` там, где он нужен.
- **403** — для указанного `projectId` и периода нет записи со статусом полного подтверждения партнёров.

**Исключение:** счёт только с **ручными** строками (`lines`), **без** `projectId`, **без** `timeEntryIds` и **без** `expenseIds` — период подтверждения не требуется (нерядкий кейс).

**Рекомендация по UX:** те же даты, что пользователь выбрал для необиллированных записей и для отправки на подпись партнёрам, передавать в `partnerBillingPeriodFrom` / `partnerBillingPeriodTo` при `POST` счёта.

---

## 2. Необиллированное время и расходы

**Запросы:**

- `GET …/invoices/unbilled-time?projectId=…&dateFrom=…&dateTo=…`
- `GET …/invoices/unbilled-expenses?projectId=…&dateFrom=…&dateTo=…`

Если для этого **`projectId`** и того же **`dateFrom`–`dateTo`** нет **полного** подтверждения партнёров, ответ **403** с текстом о необходимости сначала получить подтверждения.

**На фронте:** после **403** не показывать таблицу позиций; можно вывести то же сообщение, что в `detail`, и ссылку/переход в поток подтверждения отчёта партнёрами.

---

## 3. Список счетов при работе из биллинга

Чтобы список **не отображался** (или отображался пустым), пока нет подтверждения по выбранному периоду проекта, можно вызывать:

`GET …/invoices` с **тремя** query-параметрами:

| Параметр | Alias (camelCase) |
|----------|-------------------|
| Проект | `partnerBillingProjectId` |
| Начало периода | `partnerBillingPeriodFrom` |
| Конец периода | `partnerBillingPeriodTo` |

Если все три заданы и подтверждения **нет**, ответ:

- `items: []`
- `partnerConfirmationBlocked: true` (дополнительное поле в JSON)
- при запросе общего числа — `totalCount: 0`, если используется `includeTotalCount`

Если подтверждение **есть**, ответ обычный: список счетов по остальным фильтрам (`clientId`, `projectId`, `status`, …).

**Важно:** без этих трёх параметров список счетов ведёт себя как раньше (исторические счета видны). Тройка параметров нужна именно для экрана «счета в контексте выбранного проекта и биллинг-периода».

---

## 4. Связка с подтверждением партнёров (отчёты)

Подтверждение создаётся и подписывается через существующие маршруты отчётов, например:

- `POST …/time-tracking/reports/partner-confirmations/submit-from-preview`
- `POST …/time-tracking/reports/partner-confirmations/{id}/confirm`

У объекта подтверждения есть **`projectId`**, **`dateFrom`**, **`dateTo`**, статус до полной подписи — не `fully_confirmed`; после подписей всех партнёров проекта — **`fully_confirmed`**.

Период в **`partnerBillingPeriodFrom` / `partnerBillingPeriodTo`** при создании счёта и в запросах unbilled/list gate должен **совпадать** с этими датами.

---

## 5. Чеклист для фронта

- [x] При выборе проекта и периода для счёта сначала завершён поток подтверждения партнёров (или показывается блокировка по **403** на unbilled).
- [x] В **`POST /invoices`** для счёта с временем/расходами/ручными строками с проектом передаются **`partnerBillingPeriodFrom`** и **`partnerBillingPeriodTo`**.
- [x] Обрабатываются **403** на unbilled и **400/403** на создание счёта.
- [x] Для скрытия списка счетов без подтверждения используются три query-параметра из §3 и флаг **`partnerConfirmationBlocked`**.
